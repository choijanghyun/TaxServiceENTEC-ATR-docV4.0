## 구조적 갭 패턴 분석

### 패턴 1: Section 4.1 파이프라인의 자기완결성 부족

Section 4.1의 법인세 계산 Flow(REQ-4.1)는 (1)~(9)의 9단계로 구성되어 있으나, **다음 요소가 파이프라인 자체에 포함되어 있지 않음**:

- 연도별 세율 분기 -> Section 10.1에 위임
- 이월결손금 공제 한도/기간 -> Section 6.3에 위임
- 결손금 소급공제 경로 -> Section 6.3에 1행만
- 농어촌특별세 계산 단계 -> Section 8.2/9.1 STEP 6에 위임
- 지방소득세 계산 -> Section 9.1 STEP 8에 위임
- 최저한세 적용 범위 (4)~(6) 전체 -> 주석 위치 모호
- 감면분 추가납부세액 정의 -> Section 14에 위임
- 연쇄 재산정 메커니즘 -> 어디에도 없음

**결과**: Section 4.1만 참조하는 개발자는 불완전한 계산 엔진을 구현할 위험이 있음. Section 4.1에 핵심 파라미터와 분기를 주석으로 추가하거나, 각 단계에 "See Section X" 참조를 명시해야 함.

### 패턴 2: Section 4와 Section 9의 이중 구조

Section 4.1(법인세 계산 Flow, 9단계)과 Section 9.1(환급액 산출 프로세스, 8단계)이 유사한 계산 흐름을 다르게 기술하고 있어 혼란 가능:

| 구분 | Section 4.1 | Section 9.1 |
|------|-------------|-------------|
| 단계 수 | 9단계 ((1)~(9)) | 8단계 (STEP 1~8) |
| 농특세 | 미포함 | STEP 6 포함 |
| 지방소득세 | 미포함 | STEP 8 포함 |
| 환급가산금 | 미포함 | STEP 8 포함 |
| 세목 구분 | 법인세/종합소득세 별도 (4.1/4.2) | 통합 (분기 포함) |

**권고**: Section 4.1은 "세목별 핵심 파이프라인"으로, Section 9.1은 "환급액 산출 전체 프로세스"로 역할을 명확히 구분하고, Section 4.1에서 Section 9.1로의 연결 참조를 추가.
