## 5. 패턴 분석

### 5.1 공통 패턴: Section 1의 구조적 한계

Section 1은 **비교표(법인세 vs 종합소득세)** 형식으로 구성되어 있어, 프롬프트의 [핵심 전제]에 기술된 다음 정보를 구조적으로 수용하기 어려움:

1. **법조항 인용**: 비교표의 셀 공간 제약으로 법조항 생략 경향
2. **세부 공식**: 감면 소득 배분 공식 등 상세 수식은 비교표에 부적합
3. **개인 고유 제도**: 공동사업자, 지방소득세 별도 절차 등 법인에 대응 항목이 없는 종합소득세 고유 사항은 비교표에 포함 불가

### 6.4 구조적 개선 제안

현재 Section 1.4의 비교표 형식은 법인세/종합소득세 **공통 비교 항목**만 수용 가능하다. 종합소득세 **고유 사항**(공동사업자, 지방소득세 별도 절차 등)은 비교표에 대응 열이 없어 구조적으로 누락된다.

**제안**: Section 1.4 비교표 이후에 **"1.4.1 종합소득세 고유 특성"** 하위 섹션을 추가하여 다음을 기술:

```markdown
### 1.4.1 종합소득세 고유 특성

| 항목 | 내용 | 근거법령 |
|------|------|---------|
| 소득합산 범위 | 사업소득+근로+이자/배당+연금+기타소득 합산과세 | 소득세법 제4조 |

| 감면 소득 배분 | 산출세액 x (감면대상 사업소득/종합소득금액) x 감면율 | - |
```
